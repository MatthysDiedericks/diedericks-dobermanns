## Critical rules

- `LOGO_BASE64` must be generated by the script, not hardcoded in the prompt — the file is ~746KB base64
- The banking details (`BANKING` constant) must be a separate editable constant at the top of `generatePDF.ts` — never hardcoded inside the HTML string
- `DogLinePicker` must be inside a `ScrollView` with max height 300 — it is shown inline inside a form, not a separate screen
- After marking a dog as reserved from an invoice, confirm with an Alert before updating — never auto-update silently
- `is_payable` in expenses defaults to `false` — existing expenses are NOT affected by the migration
- When `item_type = 'dog_sale'` or `'deposit'`, show `DogLinePicker`. For `training_fee` and `other` — free-text only
- The statement must show a running balance column (cumulative) — this is calculated in-memory, not stored in DB
- All new files under 300 lines — split creditors screen into `DebtorsTab.tsx` and `CreditorsTab.tsx` components if needed
