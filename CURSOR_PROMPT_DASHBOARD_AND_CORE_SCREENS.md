### WIDGET 2 — Upcoming Heats

**Data:**
```sql
SELECT hc.*, d.name as dog_name, d.date_of_birth,
  (SELECT COUNT(*) FROM litters l WHERE l.mother_id = hc.dog_id) as litter_count
FROM heat_cycles hc
JOIN dogs d ON d.id = hc.dog_id
WHERE hc.expected_whelp_date >= CURRENT_DATE
  AND hc.status IN ('in_heat','mated','confirmed_pregnant')
ORDER BY hc.expected_whelp_date ASC
LIMIT 8
```

Display as a list:
| Expected litter date | Dam name | Dog age | # Litters | → |

- **Expected litter date** formatted: "7 Jul 2026"
- **Dam name** in gold, tappable → dog profile
- **Dog age** calculated from `date_of_birth`: "3y 1m"
- **# Litters:** previous litter count: "# Litters: 3"
- **→:** navigates to heat cycle detail

Each row has a status indicator:
- 🔴 In Heat (not yet mated)
- 🟡 Mated (awaiting confirmation)
- 🟢 Confirmed Pregnant

Header "Upcoming Heats" → navigates to `/(admin)/heats`

### WIDGET 3 — Expected Litters (mated, awaiting whelp)

**Data:**
```sql
SELECT hc.*, d.name as dam_name,
  CURRENT_DATE - hc.ovulation_date as days_since_ovulation
FROM heat_cycles hc
JOIN dogs d ON d.id = hc.dog_id
WHERE hc.status = 'mated' OR hc.status = 'confirmed_pregnant'
ORDER BY hc.expected_whelp_date ASC
```

Table columns:
| Dam | Days since ovulation | Due date | Status | → |

- **Days since ovulation:** integer (e.g. 45). If ovulation_date is null, show "—"
- **Due date** formatted with info icon: hover/tap shows "Expected: Ovulation date + 63 days"
- **Status badge:** Mated (amber) / Confirmed Pregnant (green)

### WIDGET 6 — To-Do Items

**Data:**
```sql
SELECT * FROM todo_items
WHERE is_completed = false
  AND (due_date <= CURRENT_DATE + 7 OR due_date IS NULL)
ORDER BY
  CASE WHEN due_date < CURRENT_DATE THEN 0
       WHEN due_date = CURRENT_DATE THEN 1
       ELSE 2 END,
  due_date ASC
LIMIT 8
```

Table columns:
| Due Date | Title | Linked To | ✓ |

- **Due Date:** formatted date. Red if overdue. Gold if today. White if future.
- **Title:** todo title
- **Linked To:** if `litter_id` set → show litter identifier (e.g. "🐾 Cendra Diedericks: Litter H"). If `dog_id` → show dog name. Tappable → navigates to linked entity.
- **✓ checkbox:** tapping marks `is_completed = true`, sets `completed_at`, animates row out. Optimistic update — do not wait for server.

Overdue count shown as red badge on widget header.

Header "To-Do Items" → `/(admin)/todos`

### Contract eSign Flow

When admin clicks "Send for eSign":
1. Generate a unique `esign_token` (UUID) and set `esign_expires_at = NOW() + 14 days`
2. Update `contracts.status = 'sent'`, set `esign_sent_at = NOW()`
3. Send email to buyer via Resend Edge Function with link:
   `https://diedericksdobermanns.com/sign/{esign_token}` (or app deep link)
4. Show admin: "eSign link sent to {buyer_email}. Expires {date}."

**Public signing page (web only): `app/sign/[token]/page.tsx`**

- Fetch contract by `esign_token` WHERE `esign_expires_at > NOW()` AND `status IN ('sent')`
- If token expired or not found: show "This link has expired. Please contact Diedericks Dobermanns."
- Show full rendered contract text (read-only, scrollable)
- At bottom: checkbox "I have read and agree to this contract"
- "Sign" button (only enabled after checkbox):
  - Records `signed_by_client = true`, `client_signed_at = NOW()`, `client_ip_on_sign = request IP`, `status = 'signed_client'`
  - Shows success: "Thank you. Your contract has been signed and saved. A copy will be emailed to you."
  - Sends notification to admin + email copy to buyer

**Contract PDF generation:**
Uses `jsPDF` (web) or `expo-print` (app). Renders `body_html` with merge tags replaced, wrapped in branded header/footer (DD logo, gold accents, "CONFIDENTIAL" footer). Same approach as finance PDF.
